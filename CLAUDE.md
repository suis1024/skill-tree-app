# skill-tree-app

不特定多数公開を狙う Web ゲーム。

## ジャンル
全方位シューティング × ローグライト（Brotato 型）

## ゲームデザイン

### コアループ
プレイ → 死亡 → コイン全額持ち帰り → スキルツリーで永続強化 → 再挑戦

### プレイ
- プレイヤー操作: 移動のみ（武器は自動発射）
- 1プレイの長さ: 数分〜10分
- 視点: 全方位（Brotato/Vampire Survivors 型）

### クリア条件
**10分耐久**で達成。クリア後はエンディング表示のみ（周回プレイは可）。
初回プレイは数分で死ぬ想定。メタ進行で生存時間を伸ばしていく。

### コイン
1プレイで集めたコインは死亡/クリア時に全額持ち帰り、スキルツリーの強化に使う。
プレイ中の一時強化はなし。

### スキルツリー（メタ進行）
3系統 × 各5〜7スキル = 計20スキル前後（叩き台）。複数段階アップグレード可。

- 攻撃系: 攻撃力 / 攻撃速度 / クリティカル率 / 弾の貫通 / 武器追加スロット
- 防御系: 最大HP / HP自然回復 / 移動速度 / 被ダメージ軽減 / 復活
- 経済系: コイン獲得 / 経験値 / ショップリロール / 開始時ボーナス / リトライボーナス

### マルチユーザー
なし。サーバーは進捗（コイン残高・スキル解放状態）保存のみ、ユーザーIDで管理。

## 技術スタック
- フロントエンド: React + Vite (GitHub Pages)
- バックエンド: Python + FastAPI, uv で依存管理 (Fly.io)
- DB: Neon (PostgreSQL) / ローカルは Docker Compose の Postgres

## 進捗
最小動作版（ボタンでスキル解放するだけのプロトタイプ）は実装済みだがゲーム性ゼロ。
仕様確定したので、現状のフロント/バック実装は作り直し前提。

## 設計上の注意
- 「ゲームアプリ」の指示でも、実装より先に**コアループ・面白さの仕様を確認する**
- ローグライト型なので「死ぬ → 強くなる → また挑戦」のループが中毒性の源
